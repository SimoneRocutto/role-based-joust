# UX-014: Role-based — show roles on dashboard player cards

**Priority**: P2 | **Status**: TODO | **Tags**: `NEEDS-INPUT`

## Problem

Spectators watching the dashboard during a role-based game cannot see which role each player has. The entire identity of the mode (secret roles) is invisible to the audience. Player cards show only names and "0 pts".

## Affected modes

role-based only

## Audit references

- Role-based: `05_dash_active.png` — R-D06

## Open question (NEEDS-INPUT)

**This is a gameplay design decision, not just a UI task.**

Should roles be visible to spectators on the dashboard?

1. **Always visible** — roles shown on player cards from round start. Spectators can follow the drama ("the Vampire is at low HP, will the Angel save them?"). This makes the dashboard much more engaging.
2. **Revealed on death** — roles hidden during gameplay, revealed when a player dies. Adds mystery during the round, reveals at dramatic moments.
3. **Never visible** — roles stay secret. The dashboard shows no role info. Spectators only know roles if announced via TTS.
4. **Configurable** — admin toggle in settings.

This affects spectator experience significantly. Option 1 is most spectator-friendly; option 2 is most dramatic; option 4 is safest.

## Implementation plan (after input)

1. **Client dashboard**: Add role name/icon to player cards (conditionally based on chosen visibility rule)
2. **Server**: Role data is already in the game state. May need a `revealRoles` setting if option 4 is chosen.
3. **Tests**: Unit test role display on dashboard cards.

## Dependencies

UX-003 (dashboard card redesign) — roles should be part of the new card layout.

## Estimated scope

Small if "always visible" — just add a label. Medium if conditional reveal logic is needed.
