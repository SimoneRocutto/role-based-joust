# UX-009: Death-count modes — "pts" -> "deaths" label

**Priority**: P2 | **Status**: TODO

## Problem

Death-count modes score by deaths, but the UI shows "N pts" on both dashboard player cards and phone screens. The unit should be "deaths" to match the actual scoring mechanic.

## Affected modes

death-count, death-count-teams (2/7)

## Audit references

- Summary: cross-mode issue #7
- Death-count: `05_dash_active.png` — R-D06, R-B05

## Solution

1. **Dashboard player cards**: Change "0 pts" to "0 deaths" in death-count modes.
2. **Phone screens**: Change score labels similarly.
3. The label should be mode-aware — "pts" is correct for classic/role-based modes.

## Implementation plan

1. **Server/Shared**: Add a `scoreUnit` field to the game mode config (e.g., `"pts"` for most modes, `"deaths"` for death-count).
2. **Client**: Use `scoreUnit` when rendering score labels instead of hardcoded "pts".
3. **Tests**: Unit test that death-count modes show "deaths" and other modes show "pts".

## Dependencies

None.

## Estimated scope

Small — label change with mode-aware config. Quick win.
