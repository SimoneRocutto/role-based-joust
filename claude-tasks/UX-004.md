# UX-004: Dashboard — differentiate active-round from pre-game

**Priority**: P1 | **Status**: TODO

## Problem

In 5/7 modes, the active-round dashboard is pixel-identical to the pre-game/team-selection screen. Spectators cannot tell a round is in progress. The only difference is sometimes the timer value.

## Affected modes

classic-teams, death-count, death-count-teams, long-live-the-king, domination (5/7)

Classic and role-based are less affected because they don't have a team-selection pre-game phase that looks the same.

## Solution

This will largely be solved by UX-003 (adding HP bars to player cards) — once HP bars appear during active gameplay but not during pre-game, the two phases become visually distinct. Additional measures:

1. **Phase label** — add a prominent "ROUND IN PROGRESS" or "ROUND 1 - ACTIVE" label to the dashboard header during active gameplay. The pre-game phase would show "PRE-GAME" or "TEAM SELECTION".
2. **Remove team-switching affordances** during active play (if they're still shown).
3. **Alive count becomes dynamic** — the "ALIVE: N/M" counter changing mid-round is another distinguishing signal.

## Implementation plan

1. **Client**: Add a phase label component to the dashboard header
   - Read game phase from Zustand store (already available: `gamePhase`)
   - Map phases to display labels: `pre-game` -> "PRE-GAME", `active` -> "ROUND IN PROGRESS", `round-ended` -> "ROUND OVER", etc.
   - Style prominently (large text, high contrast, visible from 3-5m)
2. **Tests**: Unit test phase label rendering for each game phase.
3. **Screenshot**: Verify that pre-game and active look different.

## Dependencies

UX-003 (HP bars) will do most of the heavy lifting. This task adds the explicit phase label as a complement.

## Estimated scope

Small — add a phase label component and wire it to existing store state.
