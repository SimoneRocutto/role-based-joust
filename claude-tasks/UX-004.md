# UX-004: Dashboard team-mode — differentiate active-round from pre-game

**Priority**: P1 | **Status**: TODO

## Context update

**Solo modes (classic, role-based) are NOT affected.** In solo modes, the dashboard uses large player cards with HP-colored borders. During active gameplay, the borders shift green→yellow→red as players take damage — this is visually distinct from the pre-game/lobby state where no HP colors are shown. The original audit flagged all modes; the real gap is team modes only.

## Problem

In team modes (classic-teams, death-count-teams, king, domination), the active-round dashboard is visually identical to the pre-game team-selection screen. Both show team panels with small player name chips, a STOP GAME button, and a status bar. Spectators cannot tell a round is in progress.

## Affected modes

classic-teams, death-count-teams, long-live-the-king, domination (4 team modes)

## Solution

This will largely be solved by UX-003 (adding HP info to team-mode chips) — once chips reflect HP during active play, the two phases become visually distinct. Additional measures:

1. **Phase label** — add a prominent "ROUND IN PROGRESS" or "ROUND 1 - ACTIVE" label to the dashboard header during active gameplay. Pre-game shows "PRE-GAME" or "TEAM SELECTION".
2. **Remove team-switching affordances** during active play (if still shown).

## Implementation plan

1. **Client**: Add a phase label component to the dashboard header
   - Read `gamePhase` from Zustand store (already available)
   - Map phases to labels: `pre-game` → "PRE-GAME", `active` → "ROUND IN PROGRESS", `round-ended` → "ROUND OVER"
2. **Tests**: Unit test phase label rendering for each game phase.

## Dependencies

UX-003 (HP on team chips) does most of the heavy lifting.

## Estimated scope

Small — add a phase label component wired to existing store state.
