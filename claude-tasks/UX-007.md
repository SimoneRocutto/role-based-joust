# UX-007: Phone team modes — show team name + team result

**Priority**: P1 | **Status**: TODO

## Problem

In all team modes, eliminated players see only individual score on their dead/round-end screen. No team name, team standing, or team result is shown. At round-end, individual rank is primary even when team result is what matters. A player can see "#1" individually while their team lost — they'd walk away thinking they won.

## Affected modes

classic-teams, death-count-teams, long-live-the-king (3 team modes)

## Audit references

- Summary: cross-mode issue #4, top priority fix #5
- Classic-teams: `17_phoneA_round_end.png` — R-P06
- King: `16_phoneA_round_end.png` — R-P06, R-B05
- Death-count-teams: (implied from team-mode pattern)

## Solution

1. **Show team name** on the phone during active play, dead screen, and round-end (e.g., "Red Team" or "Blue Team" with team color accent).
2. **Show team result at round-end** as the primary element: "Your team won!" / "Your team lost!" before individual rank.
3. **Show team score** alongside or above individual score.

## Implementation plan

1. **Client**: Modify `PlayerView` for team-mode states
   - Read `teamId` and `teamName` from store (already available from `team:update` event)
   - On round-end/game-end: fetch team standings from the `round:end` or `game:end` event data
   - Add team result as the headline above individual rank
   - Add team name badge to active and dead screens (colored pill or banner)
2. **Server**: Verify that `round:end` payload includes team scores. If not, add them.
3. **Shared types**: May need to add `teamScores` to `RoundEndPayload` if not present.
4. **Tests**: Unit test team result display. Server test that team scores are included in round-end payload.

## Dependencies

None strictly, but the round-end payload may need server changes.

## Estimated scope

Medium — client rendering + possible server payload addition. Touches 3 modes.
