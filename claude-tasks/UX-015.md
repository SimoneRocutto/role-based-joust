# UX-015: Domination — team scores as primary dashboard element

**Priority**: P1 | **Status**: TODO

## Problem

Domination is a team mode, but the dashboard shows only individual "0 pts" values with no team grouping, no team scores, and no team color coding. Spectators cannot answer "which team is winning?"

## Affected modes

domination only (but the pattern also affects death-count-teams — see UX-017)

## Audit references

- Domination: `05_dash_active.png` — R-D10, R-D06

## Solution

1. **Group players by team** on the dashboard with team-colored panels (like classic-teams and king mode already do).
2. **Show team aggregate score** prominently in each team panel header.
3. **Demote or remove individual "pts"** — in domination, the team score is what matters.

Note: This overlaps heavily with GM-001 (domination scoring rework). The scoring logic change and the UI change should happen together.

## Implementation plan

1. **Client dashboard**: Use the team-grouped layout (already implemented for classic-teams and king mode) for domination mode.
   - The dashboard likely already switches layouts based on `teamsEnabled` — verify domination sets this correctly.
   - Add team score display to team panel headers.
2. **Server**: Verify domination mode sends team scores in `game:tick` or a dedicated event.
3. **Tests**: Dashboard renders team layout for domination. Team scores display correctly.

## Dependencies

GM-001 (domination scoring rework) — the scoring logic needs to be correct before the UI can show meaningful team scores.

## Estimated scope

Small-medium if the team-grouped layout already works for other team modes. May just need to enable it for domination.
