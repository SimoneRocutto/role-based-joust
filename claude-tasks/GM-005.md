# GM-005: Team mode — score normalization for uneven teams

**Priority**: P2 | **Status**: TODO | **Tags**: `NEEDS-INPUT`

## Problem

From the user's TODO.md: "points at the end of round should exclude points made by the lowest scoring players. This doesn't happen when all teams have the same amount of players. Though, when we have 3, 2 and 2 in teams, team with 3 players will statistically score +50% points compared to other teams. So, we'll just exclude the worst player from their score count as a way to lower the disparity."

Uneven teams create scoring imbalances. A team with more players has a statistical advantage.

## Affected modes

All team modes: classic-teams, death-count-teams, long-live-the-king

## Source

User's TODO.md — team mode specific

## Open question (NEEDS-INPUT)

The user proposed excluding the worst player from larger teams. But there are alternatives:

1. **Exclude worst player** from teams with more members than the smallest team. Simple but feels punishing to individual players.
2. **Per-capita scoring** — divide team score by team size. Fair but changes the feel of scoring.
3. **Cap contributions** — each team's score counts only the top N players (where N = smallest team size). Similar to option 1 but framed differently.
4. **Weighted scoring** — smaller teams get a multiplier.

Which approach does the user prefer? The original TODO suggests option 1.

## Implementation plan (after input)

1. **Server**: Modify team score aggregation in `TeamManager` or the relevant game mode's `calculateFinalScores()`
   - Apply the normalization rule when teams have different sizes
   - Include a note in the score breakdown showing the adjustment
2. **Client**: Show the adjustment in the round-end scoreboard (e.g., strikethrough on excluded player's contribution)
3. **Tests**: Server tests for various team size combinations.

## Dependencies

## Estimated scope

Small-medium — server scoring logic change. The tricky part is communicating the adjustment clearly in the UI.
