# UX-020: Hide "ready" button on game-over in single-round modes

**Priority**: P2 | **Status**: TODO

## Problem

On the phone game-over screen, "CLICK WHEN READY" (dev mode) / shake-to-ready is shown even in single-round modes like domination. There is no next round — the game is over. This phantom action confuses players.

## Affected modes

domination (single-round). Could also affect any future single-round mode.

## Audit references

- Domination: `11_phoneA_game_over.png`, `12_phoneB_game_over.png` — R-B05

## Solution

On the game-over phone screen (`finished` state, not `round-ended`), hide the ready button entirely. Show "Game Over" without a ready-up prompt. Optionally show "Waiting for admin to start new game" or similar.

The distinction:
- `round-ended` state: show ready button (next round coming)
- `finished` state: hide ready button (game is over)

## Implementation plan

1. **Client PlayerView**: In the game-over/finished screen component, check if the game state is `finished` vs `round-ended`.
2. If `finished`: hide the ready/shake prompt. Show a static "Game Over" state.
3. If `round-ended`: keep the ready prompt as-is.
4. **Tests**: Unit test that ready button is hidden when state is `finished`.

## Dependencies

None

## Estimated scope

Small — conditional render in one component.
