# UX-024: Phone game-over: show rank instead of slot number

**Priority:** P2
**Status:** TODO
**Tags:** cross-mode

## Problem

The game-over phone screen shows the player's slot number (#1, #2) prominently, not their finishing rank. A player who finished last with 0 pts sees "#1" in huge text and could think they won.

Flagged in: role-based, classic.

## Proposed fix

Replace (or supplement) the slot number display with the player's finishing rank on the game-over screen. The rank should be derived from the final leaderboard ordering.

For example: "2nd Place" or a medal icon instead of/alongside "#1 PlayerA".

## Files likely affected

- `client/src/components/player/screens/GameOverScreen.tsx`
- Server may need to send rank in the `game:end` payload

## Notes

In team modes, rank is less meaningful than team outcome (covered by UX-007). This task focuses on solo modes.
