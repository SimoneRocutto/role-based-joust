# UX-003: Dashboard team-mode ‚Äî use full player cards during active gameplay

**Priority**: P1 | **Status**: TODO

## Decision (resolved)

Use the existing `PlayerCard` component (the solo-mode cards with HP-colored borders) inside team panels during active gameplay. Keep `CompactPlayerCard` chips for team selection only.

**Phase switching:**
- `team-selection` / `pre-game` ‚Üí team panels with compact chips (easy to follow player movements between teams)
- `active` / `round-ended` ‚Üí team panels with full player cards (HP visible via border color, dead state via skull)

This reuses the proven HP visualization (card border green‚Üíyellow‚Üíred) instead of inventing a new system for chips.

## Problem

In team modes during active gameplay, spectators see small name chips inside team panels with no HP indication. The solo-mode cards already have HP via border color, but team modes don't use them. Switching to cards during active play solves this with zero new HP visualization code.

## Affected modes

classic-teams, death-count-teams, long-live-the-king, domination (all team modes)

## Implementation plan

1. **Client dashboard**: In the team panel component (likely in `PlayerGrid.tsx` or a `TeamLobbyGrid` sub-component):
   - During `pre-game` / team-selection: render `CompactPlayerCard` chips (current behavior)
   - During `active` / `round-ended` / `finished`: render `PlayerCard` components instead
   - Cards need to be sized to fit inside team panels ‚Äî with 5-6 players per team, each card should be roughly half the width of a solo-mode card
   - Team panel containers (Red Team / Blue Team headers) stay visible to preserve team grouping
2. **Card sizing**: The `PlayerCard` component may need a `compact` or `size` prop to render at a smaller size inside team panels. Or use CSS to constrain the card width within the panel.
3. **Data**: `PlayerCard` needs `hp`, `isAlive`, `points` etc. ‚Äî verify this data is available in the team-mode player objects passed to the grid. It should be from `game:tick`.
4. **Tests**: Unit test that team panels render cards during active phase and chips during team selection.
5. **Screenshot + audit**: Capture team-mode active state to verify cards show HP borders at various damage levels.

## Dependencies

None ‚Äî `PlayerCard` component already exists and works in solo modes.

## Estimated scope

Medium ‚Äî conditional rendering based on game phase + possible card sizing work. The card component itself doesn't need changes (or minimal sizing props).
