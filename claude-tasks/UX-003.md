# UX-003: Dashboard active — add HP bars to player cards

**Priority**: P0 | **Status**: TODO | **Tags**: `NEEDS-INPUT`

## Problem

During active gameplay, the dashboard shows player name cards with no health bars, no damage indicators, and minimal alive/dead distinction. Spectators see static cards. The most exciting phase of the game produces the least visual feedback. This is the second most impactful cross-mode issue.

## Affected modes

All 7 modes

## Open question (NEEDS-INPUT)

Dashboard player card layout options:

1. **Inline HP bar** — add a colored bar inside existing player name chips. Minimal layout change.
2. **Redesigned cards** — larger cards with HP bar, player name, and optional metadata (role, deaths, team score). More informative but requires layout rework.
3. **HP as card border** — the card border color/thickness reflects HP (green thick = healthy, red thin = near death). Subtle but visible at distance.

Also: should dead players be shown differently? Options:
- Grayscale + skull overlay
- Card collapses/shrinks
- Strikethrough on name + reduced opacity

The `game:tick` event already sends full player state to the dashboard. The data is there — this is a rendering task.

## Implementation plan (after input)

1. **Client**: Modify dashboard player card component
   - File: likely in `client/src/pages/DashboardView.tsx` or a sub-component
   - Add HP bar element to each player card
   - Add dead state visual treatment
   - HP data comes from the `game:tick` event, already in the Zustand store
2. **Tests**: Client unit test for HP bar rendering at various HP levels. Test dead state rendering.
3. **Screenshot + audit**: Verify at multiple game states (all alive, some dead, all dead).
