# UX-003: Dashboard team-mode view — add HP info to player chips

**Priority**: P1 | **Status**: TODO | **Tags**: `NEEDS-INPUT`

## Context: two dashboard layouts already exist

1. **Solo modes (classic, role-based)** — large player cards with colored outlines. The outline color shifts green→red based on HP. Dead players show a skull. **This already has HP visualization.**
2. **Team modes (classic-teams, death-count-teams, king, domination)** — team panels contain small player name chips. These chips show only `#N Name` with a small colored left-border. **No HP info is visible on these chips.**

The original audit flagged "no HP bars on dashboard" across all modes — but solo modes already handle this via card border color. The real gap is the **team-mode chip layout only**.

## Problem

In team modes during active gameplay, spectators see small name chips inside team panels with no HP indication. From 3-5m on a projector, you can't tell who's healthy, who's damaged, or who's close to dying. The team mode dashboard is less informative than the solo mode dashboard during the most exciting phase.

## Screenshots showing the gap

- Classic-teams active: `11_dash_active.png` — chips with no HP
- King active: `08_dash_active.png` — chips with no HP
- Compare to classic active: `04_dash_active.png` — cards WITH HP-colored borders

## Open question (NEEDS-INPUT)

How to add HP info to team-mode player chips:

1. **Color the chip background** — same green→red gradient as solo mode card borders. Minimal layout change.
2. **Add a thin HP bar** inside or below each chip. More precise but adds visual complexity.
3. **Replace chips with mini-cards** — small versions of the solo-mode cards, inside team panels. Most informative but may not fit with many players per team.
4. **Color the left-border** of each chip (it already exists) — shift it from static team-color to HP-based color.

Also: should dead players in team mode be visually distinct? Currently the chip just stays there with no change when a player dies.

## Implementation plan (after input)

1. **Client**: Modify the team-mode player chip component to show HP state
   - HP data is already in the store from `game:tick`
   - Add visual treatment based on chosen approach
   - Add dead state visual (grayscale, skull, opacity)
2. **Tests**: Unit test chip renders HP colors at various levels. Test dead state.
3. **Screenshot**: Capture team-mode active state with damaged players.

## Dependencies

None.

## Estimated scope

Small-medium — modifying existing chip component styling. Data already available.
