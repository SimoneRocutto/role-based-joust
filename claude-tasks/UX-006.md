# UX-006: Phone dead/round-end — fix "Final Score" label + add context

**Priority**: P1 | **Status**: TODO

## Problem

When a player dies mid-round in a multi-round game, their phone shows "Final Score: N pts." "Final" implies the game is over. Players may put their phone away and miss subsequent rounds. Additionally, the dead screen shows no context about what's happening (is the round still going? will there be another round?).

The round-end phone screen is often identical to the mid-round dead screen, so players can't tell when a round actually ends.

## Affected modes

classic, classic-teams, role-based, long-live-the-king (4/7 — the modes where "Final Score" appears mid-round)

- Role-based: `08_phoneA_dead.png` — R-P02, R-P06

## Solution

1. **Rename "Final Score"** to "Score" or "Round Score" on mid-round dead screens.
2. **Add context line** below the score:
   - Mid-round dead: "Round still in progress — watch the dashboard"
   - Round-end: "Round over — shake to ready up for the next round"
   - Game-end: "Game over" (here "Final Score" is appropriate)
3. **Visually differentiate** mid-round dead from round-end:
   - Round-end could show rank (#1, #2...) and a "Get ready" prompt
   - Mid-round dead stays as the skull + "ELIMINATED" but with the context line

## Implementation plan

1. **Client**: Modify the dead/eliminated screen in `PlayerView`
   - Check `gamePhase` from store: `active` = mid-round dead, `round-ended` = round over, `finished` = game over
   - Swap label text based on phase
   - Add context line component
2. **Tests**: Unit test that correct labels appear for each phase.
3. **Screenshot**: Capture mid-round death and round-end states to verify they look different.

## Dependencies

None.

## Estimated scope

Small — label text changes + a context line. The game phase is already in the Zustand store.
