# UX-016: Domination — rework dashboard active game UI

**Priority**: P1 | **Status**: TODO

## Context update: phone is NOT the focus here

Phones are chest-mounted facing opponents. The player can't see their own screen. In domination, showing base status or zone maps on the phone makes no sense — the player can't read it, and opponents don't need that info from looking at someone's chest. Team color on the phone background is already sufficient for the phone.

**This task is now dashboard-focused only.** The user's TODO.md already listed "rework domination active game UI" as a priority.

## Problem

The domination dashboard during active gameplay shows no domination-specific information:
- No team scores
- No base ownership status
- No capture progress
- Player cards use the generic layout with no team grouping
- Status bar says "Waiting for action..." during active play

The `BaseStatusPanel` component exists but may not be visible during active gameplay.

## Solution

1. **Team-grouped layout** — use the team panel layout (like classic-teams, king mode) for domination. Group players by team with team-colored panels.
2. **Team scores** prominently in panel headers (ties into UX-015).
3. **Base status visible** — ensure `BaseStatusPanel` renders during active gameplay showing which team controls each base.
4. **Fix status bar** — show "ROUND IN PROGRESS" instead of "Waiting for action..." (ties into UX-004).

## Implementation plan

1. **Client dashboard**: Enable team-grouped layout for domination mode.
2. **Client dashboard**: Verify `BaseStatusPanel` is rendered during active phase.
3. **Client dashboard**: Add team scores to team panel headers.
4. **Tests**: Dashboard renders team layout + base status for domination.
5. **Screenshot**: Capture domination active state with bases owned.

## Dependencies

GM-001 (DONE), UX-015 (team scores), UX-004 (phase label)

## Estimated scope

Medium — dashboard layout changes. No phone work needed.
