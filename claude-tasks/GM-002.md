# GM-002: Domination — base capture by tapping team color section

**Priority**: P1 | **Status**: DONE (commit `db5809f`) | **Tags**: ~~`NEEDS-INPUT`~~

## Problem

From the user's TODO.md: "domination: allow changing base team by clicking the right color. We should have it be divided into n parts, where n is the number of teams. each of them is colored. When the red one gets pressed, red players take the base. The issue here is understanding how to still make it clear which team is currently controlling the base."

The base UI needs a rework so that each team has a tappable section, and the current owner is visually clear.

## Affected modes

domination only

## Source

User's TODO.md — top priority item

## Open question (NEEDS-INPUT)

**Visual design for base ownership clarity:**
1. **Fill/highlight approach**: The owning team's section is filled/highlighted, others are muted. Tapping a section initiates capture for that team.
2. **Border/glow approach**: The entire base has a border color matching the owning team. Sections are always visible but the owner is indicated by a surrounding glow or border.
3. **Progress bar**: When a team is capturing, a progress bar fills in their section. Once captured, that section dominates the visual.
4. **Contested state**: When capture is in progress, how to show it? Pulsing animation? Split progress bar?

The user needs to decide the visual treatment for:
- Neutral base (no owner)
- Owned base (one team controls it)
- Contested base (capture in progress)

## Implementation plan (after input)

1. **Client base view**: Redesign the `/base` page to show N colored sections (one per team)
   - Each section is tappable and sends a capture event for that team
   - Current owner is visually dominant
   - Capture progress shown during transition
2. **Server**: May need to adjust base capture logic if the tap-to-capture mechanic changes
3. **Tests**: E2e test for base capture flow. Unit test for base UI states.
