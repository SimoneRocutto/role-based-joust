# UX-001: Phone active — label the big number + show objective text

**Priority**: P0 | **Status**: TODO | **Tags**: `NEEDS-INPUT`

## CRITICAL CONSTRAINT: Phones face OPPONENTS

Phones are chest-mounted, screen facing outward. The player wearing the phone CANNOT easily see their own screen — opponents can. The phone UI is designed for people LOOKING AT the player, not the player themselves. Think of it as a jersey/badge.

This means: any info shown on the phone is public. Don't show secret roles, hidden abilities, or private strategy. DO show: player identity (number/name), team color, alive/dead status, HP (public info that adds spectator drama).

## Problem

Every mode's active phone screen is dominated by a large number (1, 2, etc.) with no label. Opponents looking at the player cannot tell if it's their rank, player number, HP, or score. This is the single most-reported CONCERN across all 7 modes (41 total CONCERNs, and this one appears in every mode).

No game objective text appears anywhere on the active phone screen. While the player can't easily read it (chest-mounted), opponents and spectators looking at the player also get no context.

## Affected modes

All 7: classic, classic-teams, role-based, long-live-the-king, death-count, death-count-teams, domination

## Open question (NEEDS-INPUT)

The big number currently shows the player's slot number (join order). Options:

1. **Remove it entirely** — free up the space for HP bar, role info, objective text
2. **Relabel as "Player #N"** — keep it but make it small and labeled, no longer the dominant element
3. **Replace with HP** — make the big number show current HP percentage instead
4. **Replace with mode-relevant metric** — rank in classic, deaths in death-count, etc.

The user needs to decide what the primary visual element on the active phone screen should be. The current player number is not useful during gameplay.

## Implementation plan (after input)

1. **Client**: Modify `PlayerView` active state rendering
   - File: `client/src/pages/PlayerView.tsx` (or the active-state sub-component)
   - Reduce/remove the giant player number
   - Add a one-line objective text (could come from the game mode config, e.g. `GameMode.getObjectiveText()`)
   - The objective text should be mode-aware: "Hold still to survive" for classic/role-based, "Hold still — fewest deaths wins" for death-count, etc.
2. **Server**: Add `getObjectiveText(): string` to `GameMode` base class, override per mode
3. **Shared types**: Add `objectiveText` to the game state payload so the client receives it
4. **Tests**: Unit test each mode's objective text. Client test that the text renders on active screen.
5. **Screenshot + audit**: Verify with `/screenshot` and `/ux-audit`

## Dependencies

None — can be done independently. Pairs well with UX-002 (HP bar) since both modify the same screen.
