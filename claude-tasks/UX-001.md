# UX-001: Phone active — consider reducing number size in non-role modes

**Priority**: P2 | **Status**: TODO | **Tags**: `NEEDS-INPUT`

## CRITICAL CONSTRAINT: Phones face OPPONENTS

Phones are chest-mounted, screen facing outward. The phone UI is for opponents/spectators looking AT the player. Think of it as a jersey/badge.

## Context

The big number on the phone active screen is the **player slot number** (join order). It serves a critical purpose in role-based mode: players receive info like "your sibling is player #3" and need to visually identify that player across the room. The number must be large and readable from distance.

The original audit flagged this as "unlabeled and opaque" — but the audit didn't understand the game's physical context. The number is intentionally large because it's a chest-worn identifier.

## Remaining question (NEEDS-INPUT)

In modes where player identification is less important (e.g., domination — you mainly care about team color, not individual identity), should the number be:

1. **Same size everywhere** — UI consistency across all modes. Simpler. The number is still useful for spectators ("player 3 just died!").
2. **Smaller in non-role modes** — free up phone screen space for mode-specific info (team score, base status, etc.). The number becomes secondary.
3. **Per-mode decision** — each mode decides how prominent the number is.

This is low priority since the current design works. Only revisit if a specific mode needs the screen real estate.

## Affected modes

Potentially: domination, death-count, death-count-teams. NOT role-based (number is critical there).

## Estimated scope

Small — CSS size change, possibly mode-conditional.
