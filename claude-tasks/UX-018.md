# UX-018: Respawn screen — label countdown + death count

**Priority**: P2 | **Status**: TODO

## Problem

The respawn screen in death-count and domination modes shows "RESPAWNING..." with an unlabeled number (countdown) and an unlabeled skull + number (death count). First-time players can't decode what any of the numbers mean. Additionally, PlayerA's respawn screen in death-count-teams has critically low contrast (red text on dark red background).

## Affected modes

death-count, death-count-teams, domination (modes with respawning)

## Audit references

- Death-count: `08_phoneA_dead.png`, `09_phoneB_dead.png` — R-P02, R-B03, R-B01
- Death-count-teams: `11_phoneA_dead.png` — R-P02
- Domination: `08_phoneA_dead.png` — R-P02, R-B03

## Solution

1. **Label the countdown**: "Respawning in 5s" or show "5" with a "seconds" label below it.
2. **Label the death count**: "Deaths: 1" instead of just skull + "1".
3. **Label the rank**: "Rank: #1" instead of just medal emoji + "#1".
4. **Fix contrast**: On team:0 (red team), use white text on dark red background instead of red-on-dark-red.

## Implementation plan

1. **Client**: Modify the respawn screen component
   - Add text labels next to each metric
   - Fix the text color for red-team respawn screen (use white or light color regardless of team)
2. **Tests**: Unit test that labels render correctly. Test contrast for both team colors.

## Dependencies

None.

## Estimated scope

Small — text label additions and a color fix. Quick win.
