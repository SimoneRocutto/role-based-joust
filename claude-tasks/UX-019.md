# UX-019: Game-over champion text — use team name in team modes

**Priority**: P2 | **Status**: DONE

## Problem

On the game-over dashboard, the footer says "Bot 1 is the champion!" even in domination mode (a team mode). The winner is the team, not an individual player. Naming an individual contradicts the team-based leaderboard shown above it.

## Affected modes

All team modes: domination, classic-team, death-count-team, long-live-the-king

## Audit references

- Domination: `10_dash_game_over.png` — R-D09

## Solution

When the game mode uses teams, the champion text should say the winning team name instead of an individual player. E.g., "Red Team wins!" instead of "Bot 1 is the champion!".

## Implementation plan

1. **Client dashboard**: In the game-over / leaderboard component, check if the current mode uses teams.
2. If teams: use the winning team name from the leaderboard data (rank 1 team).
3. If solo: keep the current individual player champion text.
4. **Tests**: Unit test champion text for team vs solo modes.

## Dependencies

None

## Estimated scope

Small — conditional text in one component.
