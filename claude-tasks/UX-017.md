# UX-017: Death-count teams — team score visibility on dashboard

**Priority**: P1 | **Status**: TODO

## Problem

In death-count-teams mode, team-level scores (aggregate death counts) are not displayed anywhere on the dashboard. The team section headers show "Red Team (1)" where "(1)" is player count, not a score. Per-player death counts are also absent from player cards. A spectator cannot determine which team is winning.

This is listed in the user's TODO.md as "death count team mode active game UI should make it easy to know which team is winning."

## Affected modes

death-count-teams only

## Audit references

- Death-count-teams: `05_dash_pregame_teams.png`, `08_dash_active.png` — R-D10, R-D06

## Solution

1. **Show team aggregate death count** in each team panel header (e.g., "Red Team — 3 deaths").
2. **Show per-player death count** on individual player cards within the team panels.
3. The team with fewer deaths is winning — make this clear visually (highlight the leading team).

## Implementation plan

1. **Client dashboard**: Add death count display to team panel headers and individual player cards.
   - Death count data should be available from `game:tick` per-player state.
   - Aggregate by team on the client side, or have the server provide team totals.
2. **Server**: Verify that per-player death counts are included in the `game:tick` payload for death-count modes.
3. **Tests**: Dashboard renders death counts. Team totals calculated correctly.

## Dependencies

UX-003 (dashboard HP bars / card redesign) — death counts should be part of the redesigned cards.
UX-009 (label "deaths" not "pts") — related label fix.

## Estimated scope

Small-medium — add death count display to existing team panel layout.
