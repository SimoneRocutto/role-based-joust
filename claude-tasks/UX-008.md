# UX-008: Phone pre-game — add "shake to ready" instruction

**Priority**: P2 | **Status**: TODO

## Problem

The shake-to-ready-up mechanic is invisible on pre-game phone screens in 4/7 modes. Players don't know they need to shake their phone to signal readiness. The game will stall until the admin force-starts.

## Affected modes

classic, role-based, death-count, domination (4/7)

Classic-teams, death-count-teams, and king mode have pre-game screens that already show some team context, but still lack the shake instruction.

## Audit references

- Summary: cross-mode issue #8, top priority fix #6
- Classic-teams: `09_phoneA_pregame.png` — R-P04, R-B03
- King: `06_phoneA_pregame.png` — R-P04

## Solution

Add a prominent "Shake to ready up!" instruction to the pre-game phone screen. Should include:
- Clear action text: "Shake your phone to ready up!"
- Ready state indicator: checkmark or "READY" badge when the player has shaken
- This should appear on ALL modes' pre-game screens, not just the 4 currently missing it

## Implementation plan

1. **Client**: Add shake instruction to the pre-game state in `PlayerView`
   - Check `isReady` from store (already tracked via `player:ready` event)
   - Show "Shake to ready up!" when not ready, "READY - waiting for others" when ready
   - Position prominently — this is the primary call-to-action during pre-game
2. **Tests**: Unit test pre-game screen shows instruction. Test ready state transition.

## Dependencies

None.

## Estimated scope

Small — add text + ready indicator to existing pre-game screen. All data already in store.
