# GM-001: Domination — no individual scores, team-only scoring

**Priority**: P1 | **Status**: DONE (commit `db5809f`)

## Problem

From the user's TODO.md: "domination: team mode should not sum players scores. Actually, there shouldn't be individual scores."

Currently, domination mode tracks and displays individual player scores ("0 pts" on cards). In domination, scoring should be team-based only — teams earn points by controlling bases, not by individual survival.

## Affected modes

domination only

## Source

User's TODO.md — top priority item

## Implementation plan

1. **Server**: Modify `DominationMode` (in `server/src/gameModes/`)
   - Remove individual point accumulation
   - Score should be team-level only, based on base control duration or base capture events
   - The `calculateFinalScores()` method needs to return team-level scores
2. **Shared types**: May need to adjust `ScoreEntry` or add a team-score variant
3. **Client**: Stop displaying individual "pts" on domination player cards
4. **Dashboard**: Show team scores only (ties into UX-015)
5. **Tests**: Server tests for domination scoring logic. Verify team scores aggregate correctly.

## Dependencies

None — this is a foundational scoring change that UX-015 and UX-016 build on.

## Estimated scope

Medium — server scoring logic change + client display changes.
