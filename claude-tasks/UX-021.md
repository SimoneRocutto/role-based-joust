# UX-021: Phone pregame — add mode description for first-time players

**Priority**: P2 | **Status**: TODO

## Problem

On the phone pregame screen, the mode name is shown (e.g., "domination") but with no explanation of what it means. A first-time player has zero context about bases, territory control, or how scoring works. They'll play domination like classic mode and miss the core mechanic.

Also: the mode name is lowercase ("domination") while the dashboard shows "Domination" — inconsistent capitalization.

## Affected modes

All modes benefit, but especially domination and other non-obvious modes (death-count, long-live-the-king, role-based).

## Audit references

- Domination: `04c_phoneA_pregame.png`, `04d_phoneB_pregame.png` — R-B03

## Solution

1. **Fix capitalization**: Use the mode's display name (from server) consistently. "Domination" not "domination".
2. **Add a one-line mode description** below the mode name on the phone pregame screen. E.g.:
   - Classic: "Last one standing wins"
   - Domination: "Control bases to earn points for your team"
   - Death Count: "Fewest deaths wins"
   - Long Live the King: "Protect your king to win"
   - Role-Based: "Each player has a secret role with unique abilities"

The description should be short enough to read at a glance — one line, no scrolling.

## Implementation plan

1. **Server/shared**: Include mode description in the pregame data sent to clients (may already be in the mode's `description` field).
2. **Client PreGameScreen**: Display the mode description below the mode name. Fix capitalization to use the mode's `name` property.
3. **Tests**: Unit test that description renders for each mode.

## Dependencies

None

## Estimated scope

Small — add one line of text to the pregame screen + capitalize mode name.
